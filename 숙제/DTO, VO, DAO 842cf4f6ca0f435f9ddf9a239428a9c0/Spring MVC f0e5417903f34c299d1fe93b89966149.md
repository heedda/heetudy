# Spring MVC

### spring 어플리케이션을 개발할 때 사용한다.

MVC의 약자는 Model-View-Controller이다

- Model : 어플리케이션 계층의 정보, 데이터 의미
    - 단지 로직만 처리한다.
- View : 화면 출력 로직을 담당
    - 사용자의 인터페이스가 표현되는 공간이다.
- Controller : Model과 View를 연결하는 제어 로직을 담당
    - Model과 View를 연결해준다.
        - 로직이란 : 논리적 흐름을 이야기 한다. 
         한마디로 논리

그래서 MVC는 애플리케이션을 개발할 때 사용하는 디자인 패턴이다.
